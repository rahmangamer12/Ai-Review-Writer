💻 Section 1: C++ Programming Fundamentals
Q: Explain the basic structure of a C++ program. ▼

A C++ program consists of 3 main parts:

  1. Header Files: They contain pre-written functions. e.g., #include <iostream> is used for input/output.
  2. Namespace: using namespace std; tells the compiler to use standard library names.
  3. Main Function: int main() is the starting point of program execution.
#include <iostream>
using namespace std;

int main() {
    cout << "Abdul Rehman will pass!" << endl;
    return 0;
}
Q: What are the rules for naming a variable? ▼
  • It can only contain letters, digits, and underscores (_).
  • It cannot start with a digit. (e.g., int 1stName; is wrong).
  • No spaces are allowed. (e.g., int my age; is wrong).
  • Keywords (reserved words like int, float, if, else) cannot be used as variable names.
Q: Difference between while loop and do-while loop? (V. IMP) ▼
While Loop Do-While Loop
Condition is checked BEFORE executing the body (Entry-controlled). Condition is checked AFTER executing the body (Exit-controlled).
If condition is false initially, the loop will never execute. The loop will execute at least once, even if the condition is false.
📦 Section 2: Data Structures (Stack & Queue)
Q: Define Queue and Stack. Write their differences. ▼

Stack: It is a linear data structure that works on the LIFO (Last In First Out) principle. Example: A stack of plates. Jo plate sab se aakhir mein rakhi, wo sab se pehle uthayi jayegi.

Queue: It is a linear data structure that works on the FIFO (First In First Out) principle. Example: A line of people at a ticket counter. Jo pehle aayega, usay pehle ticket milegi.

Key Terms:
• PUSH: Inserting an element into a Stack.
• POP: Removing an element from a Stack.
• Enqueue: Inserting an element into a Queue.
• Overflow: Condition when we try to add an element to a Stack/Queue that is already FULL.
⚠️ Section 3: Translators & Errors
Q: Define Compiler and Interpreter. ▼

Both are Language Translators that convert High-Level Language (Human readable) into Low-Level Language (Machine code 0s and 1s).

  • Compiler: Reads the entire program at once, translates it, and generates an object code. It is faster but debugging is harder.
  • Interpreter: Translates the program line-by-line. If it finds an error, it stops immediately. Slower, but debugging is easier.
Q: What are errors? Explain Syntax and Logical Errors. ▼

Bugs or mistakes in a program are called errors.

  1. Syntax Error: Mistake in the grammar or rules of the programming language. Example: Missing a semicolon ; at the end of a line. (Caught by compiler easily).
  2. Logical Error: Mistake in the formula or logic of the programmer. Program runs fine but gives the wrong output. Example: Writing a - b instead of a + b. (Hardest to find).
  3. Runtime Error: Program crashes during execution. Example: Dividing a number by zero.

Q: Define Algorithm. Write its advantages. ▼

Definition: A step-by-step written procedure in simple English to solve a specific problem.

Advantages:
  • Easy to understand as it is written in simple English.
  • Helps to find and fix errors before writing actual code.

🔌 Section 5: Logic Gates & Scratch Basics
Q: Define AND, OR, NOT and NOR Gates. ▼
  • AND Gate: Performs Logical Multiplication (.). Output is 1 ONLY if all inputs are 1.
  • OR Gate: Performs Logical Addition (+). Output is 1 if AT LEAST ONE input is 1.
  • NOT Gate (Inverter): Reverses the input. If input is 0, output is 1. If input is 1, output is 0.
  • NOR Gate: Opposite of OR Gate (OR + NOT). Output is 1 ONLY if all inputs are 0.

*Truth Table: A mathematical table used in logic to compute the functional values of logical expressions.

Q: Explain Scratch terms: Script, Sprite, Backdrop. ▼
  • Sprite: A character or object in Scratch that performs actions on the stage (e.g., the default Cat).
  • Script: A stack of blocks joined together to tell a Sprite what to do (like a program).
  • Backdrop: The background image of the Stage where sprites move.
Blocks Purpose:
  • forever: Runs the blocks inside it over and over again infinitely.
  • go to x: y:: Instantly moves the sprite to specific coordinates on the screen.
