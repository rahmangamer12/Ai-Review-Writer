📘 All 15 Short Questions (From Your Picture)

1. Describe local variables used in C++?

Variables declared inside a specific block or function `{ }` are called local variables. They can only be accessed and used within that specific block. They are destroyed when the block ends.

2. Describe translator and why we need a language translator?

A translator is a system software that converts High-Level Language (English-like code) into Low-Level Language (Machine code 0s and 1s). We need it because computers cannot understand human languages or high-level code directly.

3. Importance of flowchart for solving a problem and advantages of algorithm & flowchart?

Importance: It provides a visual map of the problem, making complex logic easy to understand.
Advantages of Algorithm: Easy to write, independent of programming syntax, helps find logical errors early.
Advantages of Flowchart: Visual representation is easier to grasp, helps in debugging, and acts as good documentation.

4. Enlist 5 high-level languages & Rules for naming variables in C?

5 Languages: 1. C++ 2. Java 3. Python 4. C# 5. PHP.
Rules for naming variables: 1. Must start with an alphabet or underscore (_). 2. Cannot contain spaces. 3. Cannot use reserved keywords (like int, if). 4. It is case-sensitive.

5. Describe script area of Scratch Editor?

The script area is the large empty space on the right side of the Scratch editor. This is where programmers drag, drop, and assemble coding blocks to create scripts that control the sprites.

6. Simplify Boolean Algebra Expression: Z = AB + A(B+C) + B(B+C)

Step 1: Multiply inside brackets -> Z = AB + AB + AC + BB + BC
Step 2: Rule AB + AB = AB, and BB = B -> Z = AB + AC + B + BC
Step 3: Take B common from (B + BC) -> B(1+C). Since 1+C = 1, it becomes B.
Step 4: Now expression is -> Z = AB + AC + B
Step 5: Take B common from AB + B -> B(A+1) = B.
Final Answer: Z = B + AC

7. Write any 3 key benefits of IDE (Integrated Development Environment)?

1. Syntax Highlighting: It highlights keywords in different colors making code easy to read.
2. Auto-completion: Suggests variable names and syntax, saving time.
3. Built-in Debugger: Helps find and fix errors easily in one place.

8. Define logical operators with examples?

Logical operators evaluate multiple conditions and return True or False.
1. AND (&&): True only if ALL conditions are true. (e.g., A > 5 && B < 10)
2. OR (||): True if ANY ONE condition is true. (e.g., A == 1 || B == 2)
3. NOT (!): Reverses the result. (e.g., !(A == B))

9. Write short notes on: Strings in C++, Switch statements, Push & Pop, Cursor tools

Strings in C++: Used to store text/words. Written in double quotes (e.g., string name = "Ali";).
Switch Statement: A multiple-choice selection statement. It checks a variable against many 'cases' and runs the matching code.
Push & Pop: Operations used in a Stack. 'Push' inserts an item at the top. 'Pop' removes the item from the top.
Cursor tools: Tools in software (like Scratch) used to interact with the interface (Select, Duplicate, Delete, Grow, Shrink).

10. Why we make block of statements using braces {}?

Braces { } group multiple statements together into a single logical block. This is required in loops (like for/while) or conditions (like if/else) so the compiler knows which group of lines to execute together.

11. Purpose & syntax of do-while, cout, switch, break?

do-while: Runs code at least once before checking condition. Syntax: do { //code } while(condition);
cout: Used to print output on screen. Syntax: cout << "Hello";
switch: Selects one block of code from many. Syntax: switch(var) { case 1: //code break; }
break: Used to immediately exit a loop or switch statement. Syntax: break;

12. 3 advantages of user-defined functions & why we use header files?

Advantages of Functions: 1. Code Reusability (write once, use many times). 2. Makes programs shorter and cleaner. 3. Easier to find and fix errors.
Header Files: Used because they contain pre-written built-in functions (like cout in iostream). We include them to use those ready-made features.

13. What is meant by syntax error? Name 3 frequent syntax errors.

Syntax error occurs when grammatical rules of C++ are violated. The program will not compile.
3 Frequent Errors: 1. Forgetting a semicolon (;) at the end of a statement. 2. Missing curly braces { }. 3. Spelling mistake in keywords (e.g., 'mainn' instead of 'main').

14. Differentiate the following:

i. Low-Level vs High-Level: Low-level is machine code (0s,1s), hard for humans. High-level is English-like (C++, Java), easy for humans.
ii. Constant vs Variable: Constant's value cannot change during execution. Variable's value can change.
iii. NAND vs NOR Gate: NAND is opposite of AND (output 0 only when all inputs 1). NOR is opposite of OR (output 1 only when all inputs 0).
iv. Source Code vs Object Code: Source code is written by human in High-level language. Object code is the machine language generated by the compiler.
v. Stack, Queue & Tree: Stack is LIFO (Last In First Out). Queue is FIFO (First In First Out). Tree is a non-linear hierarchy (parent-child).

15. Define the following definitions:

i. Computer Program: A set of step-by-step instructions given to a computer to perform a specific task.
ii. Algorithm: A step-by-step logical procedure written in simple English to solve a problem.
iii. Jump Statement: Used to alter the normal flow of a program (e.g., break, continue, goto).
iv. Sprites: The characters or objects in Scratch that perform actions based on code.
v. Inverter: Another name for the NOT gate. It reverses the input (0 becomes 1, 1 becomes 0).

📙 All 6 Long Questions (From Your Picture)

1. What is Data Structure? Describe its types, Explain classification of linear & non-linear structures?

Data Structure: It is a way of organizing, storing, and managing data in computer memory so that it can be accessed and updated efficiently.

Types of Data Structures:
1. Linear Data Structure: Data elements are arranged in a sequential or straight line. Each element is connected to its previous and next element. Examples:
- Array: Stores elements of the same data type in continuous memory.
- Stack: Works on LIFO (Last In First Out).
- Queue: Works on FIFO (First In First Out).

2. Non-Linear Data Structure: Data elements are NOT arranged in a sequence. They form a hierarchical structure (branches). Examples:
- Tree: Nodes connected like branches, having a root node and child nodes.
- Graph: A network of interconnected nodes.

2. What is translator? Types. Why needed? 5 high-level languages. Define loop, its types and nested loop.

Translator & Need: It is software that converts High-Level Language into Low-Level machine language. We need it because computers only understand 0s and 1s.
Types of Translators: 1. Compiler (translates whole program at once). 2. Interpreter (translates line-by-line). 3. Assembler (for assembly language).
5 High-Level Languages: C++, Java, Python, C#, Pascal.

Loop & Types: A loop is a structure that repeats a block of code until a condition is met.
1. For Loop: Used when the number of iterations is known.
2. While Loop: Used when iterations are unknown; checks condition first.
3. Do-While Loop: Executes code at least once before checking condition.
Nested Loop: Writing one loop inside the body of another loop is called a nested loop.

3. What are operations? Describe C++ operators. Write program for Arithmetic Assignment. Differentiate Arithmetic & Relational.

Operations/Operators: Operators are special symbols used to perform calculations or logical operations on variables and values.
Types in C++: Arithmetic (+, -, *, /), Relational (==, >, <), Logical (&&, ||, !), Assignment (=).

Difference:
- Arithmetic Operators: Perform mathematical calculations. Output is a number. Example: 5 + 3 = 8.
- Relational Operators: Compare two values. Output is True or False (1 or 0). Example: 5 > 3 is True.

Program using Arithmetic Assignment: 

#include <iostream>
using namespace std;
int main() {
    int x = 10;
    x += 5; // Same as x = x + 5 (Addition Assignment)
    cout << "Value of x is: " << x; // Output will be 15
    return 0;
}

4. Define error? Different types with examples. Draw flowchart symbols. Differentiate Algorithm & Flowchart.

Error (Bug): A mistake or flaw in a computer program that produces an incorrect or unexpected result.
Types of Errors:
1. Syntax Error: Grammatical mistake (e.g., missing semicolon).
2. Logical Error: Mistake in human logic (e.g., writing + instead of - in a formula). The program runs but gives the wrong answer.
3. Runtime Error: Code crashes during execution (e.g., dividing a number by zero).

Algorithm vs Flowchart:
- Algorithm: Step-by-step instructions written in simple English text. Harder to visualize but easy to type.
- Flowchart: Graphical/visual representation of the algorithm using shapes. Easy to understand visually.

Note for Exam: Remember to draw symbols: Oval (Start/Stop), Parallelogram (Input/Output), Rectangle (Process), Diamond (Decision).

5. Define problem? How problem can be solved. Explain in detailed.

Problem: In computer science, a problem is a specific task or challenge that needs to be solved using computer programming.
Steps to solve a problem (Problem Solving Process):
1. Defining the Problem: Carefully read and understand exactly what is required.
2. Analyzing the Problem: Identify the inputs needed, the processing required, and the desired output.
3. Planning the Solution: Write an Algorithm or draw a Flowchart to map out the logic step-by-step.
4. Coding (Writing the Program): Translate the planned algorithm into a programming language like C++.
5. Testing and Debugging: Run the program with sample data to check for errors (bugs) and fix them.
6. Documentation: Write notes inside the code so others can understand it in the future.

6. Define truth table. Purpose. Construct tables & Simplify Expressions.

Truth Table & Purpose: A mathematical table used in logic to show all possible combinations of inputs and their exact outputs. Its purpose is to test and verify if a boolean expression or digital circuit is working correctly for all scenarios.

(a) Table for (A+B)(A+C) = A+BC
You need 3 columns A, B, C (000 to 111). Evaluate LHS and RHS separately, both columns will match perfectly.
(b) Table for Z = (A+B)(A+B)
Since (A+B) AND (A+B) is just A+B. The output column Z will be identical to an OR Gate output.
(c) Y = A.B.C -> Only the last row (1,1,1) will output 1. All other rows output 0.
(d) Y = A+B+C -> Only the first row (0,0,0) will output 0. All other rows output 1.

Simplifying Boolean Expressions:
1) Z = (A.B) + (A+B)'
Apply De Morgan's on 2nd part: (A+B)' = A'.B'
Z = AB + A'B' (This is the formula for XNOR Gate).
2) Z = (A+B) + (A.B)'
Apply De Morgan's: (A.B)' = A' + B'
Z = A + B + A' + B' -> Reorder: (A + A') + (B + B') = 1 + 1 = 1. Output is always 1.
